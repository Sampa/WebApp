Functional description

Description

Manages the crane's fork extension and retraction mechanism based on a discrete integer command. It handles the physical output assignments to the crane hardware, reads sensor feedbacks to determine position, and includes a built-in safety watchdog timer. If the forks take longer than the specified tTimeoutDuration to reach their destination, a physical jam error is raised and all movements are aborted.

Interface Specifications

  • tTimeoutDuration: Time input specifying the maximum safe travel duration (default: 6s).
  • iCmd: Integer input commanding the forks (0=Stop, 1=Extend Entry, 2=Extend Exit, 3=Retract).
  • xAtEntry: Boolean output indicating forks are fully extended toward the entry side.
  • xAtMiddle: Boolean output indicating forks are fully retracted in the safe travel center position.
  • xAtExit: Boolean output indicating forks are fully extended toward the exit side.
  • xErrorJam: Boolean output indicating a physical jam or sensor fault due to timeout.
  • ioCrane: In/Out reference to the crane data structure (typeCrane).
  • ioInput: In/Out reference to the main input structure (typeMainInput).

Safety Interlocks

  • A timeout watchdog instTravelTimer asserts xErrorJam if movements take longer than allowed.
  • When xErrorJam is active, all fork extension outputs are forced FALSE to prevent hardware damage.
  • The fault is only clearable by issuing a STOP command (iCmd = 0).
