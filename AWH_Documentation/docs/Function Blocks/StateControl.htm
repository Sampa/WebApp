Functional description

Description

Implements a clean state machine with strict safety interlocking, followed by a centralized SPOW (Single Point of Write) output mapping. Manages Auto, Jog, Stopped, Emergency, and Reset modes. Evaluates emergency stop inputs and forces the crane to freeze its target position while safely managing the fork extension.

Interface Specifications

  • ioInput: In/Out reference to the main system inputs (typeMainInput).
  • ioOutput: In/Out reference to the main system outputs (typeMainOutput).
  • ioState: In/Out reference to the central machine state structure (typeMachineState).
  • ioScada: In/Out reference to SCADA controls (typeScada).
  • ioCrane: In/Out reference to the crane data structure (typeCrane).

Safety Interlocks

  • Emergency Stop immediately forces MachineStateEmergency and retracts forks or stops crane.
  • Any transition out of MachineStateAuto or MachineStateJog requires a new Reset sequence.
  • Actuators are entirely hardware-disabled during Stopped and Emergency states.
