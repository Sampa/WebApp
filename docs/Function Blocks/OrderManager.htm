Functional description

Description

Intercepts automatic warehouse operations when an external SCADA system requests a specific number of items of a particular color. Validates the request against available stock (ioWarehouseData), coordinates prioritized unloading with the main sequence, and implements a safety watchdog timer to abort and signal an error if the order execution hangs.

Interface Specifications

  • xOrderRequest: Boolean input. A rising edge triggers the evaluation of the new order.
  • iRequestedType: Integer input for item color/type (1 = Blue, 4 = Green, 7 = Black).
  • iRequestedQty: Integer input specifying the number of items requested.
  • xSequenceIdle: Boolean input indicating the main state machine is idle and can accept the order intercept.
  • xItemUnloadedPulse: Boolean input pulse from the exit conveyor when an item leaves.
  • iMachineState: Integer input pointing to the current machine state for safety aborts.
  • xOrderOverrideActive: Boolean output that commands the main sequence into UNLOAD mode.
  • iForcedUnloadType: Integer output that forces the unload algorithm to search for the requested type.
  • xOrderError: Boolean output triggering a SCADA alert for invalid orders or timeouts.
  • ioWarehouseData: In/Out reference to the warehouse rack data structure.

Safety Interlocks

  • Order is immediately aborted if iMachineState indicates Emergency or Stopped.
  • Request is rejected (Error) if requested quantity exceeds available stock or is <= 0.
  • Watchdog aborts active orders taking longer than ORDER_TIMEOUT_LIMIT (3m).
